## bonus
### 1.
使用sha256算法哈希，盐来自于phpMyAdmin创建用户页面自动生成的一个密码，保证根据哈希值猜不出盐和密码。

### 2.
在前述服务器上部署了网站，没有申请域名。域名展示只限本地。没有HTTPS加密。

### 3.
使用了jQuery。

原生JavaScript的DOM操作非常烦琐，而jQuery对此提供了多合一、名称简短的各种函数操作，算是作为语法糖了。

另外，jQuery的AJAX操作比原生JS提供的要简便、直观得多。这样可以有一部分代码实现前后端分离，后端只要输出一个JSON数组或其他原始数据，交给前端展示。也避免页面反复刷新。

### 4.
使用了Vue。

对于动态生成的内容，用Vue进行插值、绑定，比纯PHP和JS代码的实现更清晰、简洁。这样在前端设计时可以不必关注具体的实现，而且Vue把事情都做了，后期我也不必自己补充太多。特别是v-for用起来尤其方便，在本次PJ的动态图片展示中好处体现得很明显。

### 5.
Vue的基本思想是组件，这为我提供了一个全新的思路。在我们课内所讲内容中，HTML、CSS、JS应当各自内聚，而互相之间联系松散。而组件的思路正好相反，组件内各种语言的内容高度耦合，组件与组件之间松散地组成一个页面。这是因为一个组件作为一个单元，它的结构、功能、样式是密不可分的。

受此启发，我用本地PHP写了一个编译器，将页面拆成多个组件来写，最后编译成完整的页面。使用数据库进行文件管理。用PHP文件完全取代HTML文件，它们之间的引用会自动引入相应的CSS、JS文件。

这样的设计对于页面之间有差别但页面内运行时为静态的HTML内容特别有效，实现了代码复用。CSS文件通过class值，也可以自由关联其他文件。

但是对JS的代码复用就很困难，不同的页面用的JS代码不完全一样，也有互相关联，但是我编译时自动给独立的文件加上了闭包，因而无法有效代码复用。对于动态内容的处理也明显不如Vue方便。（未使用Vue组件）
